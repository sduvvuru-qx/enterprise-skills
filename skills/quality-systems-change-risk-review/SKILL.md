# quality-systems-change-risk-review

Use when the task is specifically change-driven review under DRBFM.

## When to Use

- DRBFM entry creation
- change point review
- changed -> concern -> countermeasure reasoning

## Workflow

1. State the exact change.
2. Identify what could go wrong because of the change.
3. Propose countermeasure.
4. Define how the countermeasure will be evaluated.

## Key Decisions

- anchor DRBFM on the change, not generic risk
- keep concern distinct from mitigation
- show evaluation path explicitly
