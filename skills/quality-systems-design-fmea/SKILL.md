# quality-systems-design-fmea

Use when the task is specifically inside DFMEA, not just generic quality-form work.

## When to Use

- DFMEA entry generation
- severity / occurrence / detection scoring
- RPN reasoning

## Workflow

1. Define function and failure mode.
2. Write effect and cause clearly.
3. Score S/O/D and calculate RPN.
4. Add controls, actions, and owner/date logic.

## Key Decisions

- keep failure mode separate from cause
- score from risk logic, not intuition-only
- tie actions to a concrete mitigation
