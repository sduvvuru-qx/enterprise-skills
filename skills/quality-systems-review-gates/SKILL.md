# quality-systems-review-gates

Use when the work is about design/test review readiness under review-gate logic.

## When to Use

- design review readiness
- test review readiness
- DR/TR gate framing

## Workflow

1. Identify the gate or review being prepared.
2. Check what evidence and artifacts should already exist.
3. Flag readiness gaps.
4. Return a gate-focused readiness view, not a generic summary.
