# Examples and Phrase Bank

## Current Bryan / Yingying AI tooling thread

### Better version

Use when replying in the existing RFQ AI thread. Let Outlook signature handle the close.

```text
Hi Bryan and Yingying,

Adding my thoughts here so we keep this in the same thread.

I agree with Bryan that anything involving RFQ or program data needs to stay within approved company guidelines.

One thing I would separate is the type of AI support we are talking about. M365 Copilot is useful for Office, Teams, Outlook, and summaries, but it may not cover some engineering workflows like scripts, local files, Git work, or repeatable automation.

If IT wants us to stay within the Microsoft/GitHub environment, GitHub Copilot CLI may be closer to that engineering workflow.

The open question may be what approved path IT wants engineering to use for these cases: M365 Copilot only, GitHub Copilot / Copilot CLI, Azure / Graph API access, or another approved enterprise tool.

I can pull together the current request status and a short comparison if that helps.
```

### Why it works

- Keeps the thread context.
- Avoids heavy ticket dump.
- Clarifies the engineering distinction without sounding like a tool sales pitch.
- Leaves an actionable offer.
- Does not duplicate the Outlook signature.

## Common rewrites

### Too formal

```text
I would like to take this opportunity to clarify the broader context regarding our various AI tooling requests.
```

Better:

```text
One thing I would clarify is that the requests cover a few different needs.
```

### Too vague

```text
Let me know if you need any updates.
```

Better:

```text
Bottom line: the item is still open, and the next blocker appears to be approval/routing. I can send the request details if helpful.
```

### Too memo-like for a thread

```text
Maybe the better question for IT is what path they want engineering to use:
1. M365 Copilot only
2. GitHub Copilot / GitHub Copilot CLI
3. Azure / Graph API access
4. Enterprise tools like Claude or OpenAI, if approved
```

Better:

```text
The open question may be what path IT wants engineering to use: M365 Copilot only, GitHub Copilot / Copilot CLI, Azure / Graph API access, or another approved enterprise tool.
```

## Useful Sabarish phrases

- `Adding my thoughts here so we keep this in the same thread.`
- `One thing I would separate is...`
- `The open question may be...`
- `I can pull this together if helpful.`
- `Current understanding is...`
- `Bottom line: ...`
- `I think the risk is...`
- `This may need IT / QA / JP confirmation before we treat it as closed.`
- `I will hold unless you prefer I follow up directly.`
- `Does this direction look OK?`

## Phrases to avoid

- `I hope this email finds you well`
- `I wanted to reach out regarding`
- `It is important to note`
- `Furthermore`
- `Moreover`
- `In order to`
- `Please find below`
- `Leverage`
- `Robust solution`
- `Comprehensive approach`
- `Seamless integration`
- `Optimize productivity`

## Greeting patterns

- Two named people: `Hi Bryan and Yingying,`
- One person, normal: `Hi Bryan,`
- Group thread, everyone active: `Hi all,`
- Japanese colleague: `Hayakawa-san,` or `Hi Hayakawa-san,`
- Customer/supplier: use the name and keep it formal enough for forwarding.

## Ending patterns

When Outlook signature is present, end on the last content sentence:

```text
I can pull together the current request status and a short comparison if that helps.
```

When no signature is present:

```text
Thanks,
Sabarish
```
