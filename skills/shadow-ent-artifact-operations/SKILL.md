# shadow-ent-artifact-operations

Use when the end product is an artifact rather than only an answer.

## When to Use

- deck or slide work
- spreadsheet output
- template selection
- customer-safe or management-safe packaging

## Workflow

1. Identify artifact type and audience.
2. Choose the right branded or standard base.
3. Reduce source material into decision-ready content.
4. Keep internal notes out of finished external output.

## Key Decisions

- **Internal working artifact**: allow more caveats and open questions.
- **Customer-safe artifact**: only include evidence-backed claims and cleaned language.
- **Template missing**: use the safest standard/branded fallback instead of building from blank.

## Output Shape

- purpose
- audience
- content blocks
- unresolved items that must stay internal

## Gotchas

- Letting work-log tone leak into review material.
- Overloading artifacts with raw evidence instead of reduced findings.
- Picking a template after content is already written.
