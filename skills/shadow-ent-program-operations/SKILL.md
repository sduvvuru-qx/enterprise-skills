# shadow-ent-program-operations

Use when the task needs program-aware context before deeper engineering or management work.

## When to Use

- program status
- fresh context
- preparing for reviews or updates
- cross-program ambiguity

## Workflow

1. Identify target program.
2. Gather current signals from approved enterprise sources.
3. Separate stable facts, fresh updates, and unresolved gaps.
4. Return the brief needed for the next decision.

## Key Decisions

- **Program ambiguous**: state the ambiguity and narrow by artifact, meeting, or stakeholder context.
- **Freshness uncertain**: mark the update with the source date instead of presenting it as current fact.
- **Cross-program dependency exists**: show the relationship without collapsing the programs together.

## Output Shape

- program
- current state
- major open issues
- key stakeholders
- next check or decision

## Gotchas

- Mixing historical context with the latest status.
- Treating one mail thread as full program truth.
- Overexplaining background when the user only needs the next move.
